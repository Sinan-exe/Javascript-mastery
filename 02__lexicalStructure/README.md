# Lexical Structure of JavaScript

The lexical structure of a programming language is the set of fundamental rules of how you write the progamming langauge and it is a lowest level syntax of a language

# What this is 

It explains how javascript code is read before execution

# why this matters

Before javaScript code runs, it must parse it there are certain rules for lexical structure :
- Digits are not allowed as the first character in Identifiers(it is a name used to identify a variables, functions, classes etc..)
- Cannot use Reserved Words as Identifier except(from, target ans set)
- the unicode escapes begins with "\u"
- javaScript treats the line break as semicolon until and unless it cannot parse the next line of code
